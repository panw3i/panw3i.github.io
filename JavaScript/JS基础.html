廖雪峰JS笔记

简介

  • ECMAScript是一种语言标准，而JavaScript是网景公司对 ECMAScript标准的一种实现。

  • script默认的 把 type 指定为JavaScript。

  • 开发者工具的“源码(Sources)”，掌握断 点、单步执行等高级调试技巧

语法

  • 浏览器中负责执行 JavaScript代码的引擎会自动在每个语句的结尾补上 ;
  • JavaScript严格区分大小写，如果弄错了大小写，程序将报错或者运行不正常。

数据类型和变量

  • JavaScript不区分整数和浮点数，统一用Number表示
  • 字符串是以单引号'或双引号"括起来的任意文本
  • 布尔值和布尔代数的表示完全一致，一个布尔值只有 true 、 false 两种值
  • JavaScript允许对任意数据类型做比较
  • == 比较，它会自动转换数据类型再比较，很多时候，会得到非常诡异的结果,不要使用 == 比较，始终坚持使用=== 比较
  • NaN === NaN
  • 浮点数不要用于比较
  • undefined 仅仅在判断函数参数是否传递的情况下有用

数组

  • 数组是一组按顺序排列的集合，集合的每个值称为元素。JavaScript的数组可以包括任意数据类型
  • 强烈建议直接使用[]
  • 数组索引超出了范围，返回undefined
  • 直接给Array的length赋一个新值,会导致Array的大小变化
  • 索引超过范围,也可引起Array的大小变化

对象

  • JavaScript的对象是一组由键-值组成的无序集合
  • JavaScript对象的键都是字符串类型，值可以是任意数据类型。
  • 访问属性是通过.号操作符完成的,但这要求属性名必须是一个有效的变量名,如果属性名中包含特殊字符,就必须要用''
  • js的对象是动态类型,可以动态的给一个对象添加属性和删除属性
  • 如果访问一个不存在的属性,不会报错,返回undefined
  • 检测属性是否拥有一个属性,使用in,属性是如果是继承也会返回true
  • 检测对象自身拥有的属性

变量

  • 变量名是大小写英文、数字、 $ 和 _ 的组合，且不能用数字开头。

  • 变量可以反复赋值，而且可以是不同类型的变量，但是要注意只能
    用 var 申明一次

  • 如果一个变量没有通过 var 申明就被使用，那么该变量就自动被申明为全局变量：

  • 使用 var 申明的变量则不是全局变量，它的范围被限制在该变量被申明的函数体内（函数的概念将稍后讲解），同名变量在不同的函数体内互不冲突。

  • 启用strict模式的方法是在JavaScript代码的第一行写上use strict

字符串

  • \u###表示一个Unicode字符
  • 获取字符串某个指定位置的字符，使用类似Array的下标操作，索引号从0开始
  • 数组索引超出范围的索引不会报错，但一律返回undefined
  • 字符串是不可变的，如果对字符串的某个索引赋值，不会有任 何错误，但是，也没有任何效果
  • 方法本身不会改变原有 字符串的内容，而是返回一个新字符串
  • indexOf()会搜索指定字符串出现的位置,没有返回-1
  • 判断一个是属性是否是对象自身的属性,使用hasOwnProperty()
  • for...in用于遍历对象的key

新增加的数据类型

  • 初始化map需要一个二维数组,或者直接初始化一个空Map
var m = new Map([['Michael', 95], ['Bob', 75], ['Tracy', 85]]);
  • Map增删改查及成员关系
 var m = new Map();
 m.set('Adam',67);
 m.has('Adam');
 m.get('Adam');
 m.delete('Adam');

  • Map一个key只能对应一个value,后定义的值会覆盖前面定义的值

  • set的定义

var s1 = new Set();
var s2 = new Set([1,2,3]);
  • 重复的元素在Set中自动被过滤
var s2 =  new Set([1,2,2,3,'3']);
s;  // Set {1,2,3,"3"};
  • 通过add(key)进行Set的元素添加
  • 通过delete(key)进行Set的元素删除

  • Map和 Set是ES6新增加的数据类型

  • 可迭代对象 : Array Map Set

  • 使用for...of 对集合进行遍历

  • for...in可迭代对象所有的属性, for...of只迭代集合本身的元素

  • 数组的回调函数依次是 element ,key ,array

  • JavaScript的函数调用不要求参数必须一致，因此 可以忽略它们

  • var a = ['A','B','C']
a.forEach(function (element,index,array){
    // element :指向元素的值;
    // index :当前的索引;
    // array : 指向Array对象本身;        
}
/*  a.forEach(function (element){};
*/

  • Map的回调函数依次是 value key map

函数

* 函数中如果没有return 语句 函数执行完毕后也会返回结果 ,只是结果为undefined
* 函数名是指向该函数的变量

* js 允许传入任意个数的参数而不是影响调用,因此传的参数比定义多也没有问题,虽然函数内部可能并不需要这些参数 
* 传的参数比定义的少也没有问题
* arguments只在函数内部起作用,而且永远指向当前函数的调用者传入的参数,它是一个类Array对象
* 即使函数没有定义做任何参数,arguments对象依然存在
* rest参数只能写在最后,rest标识前面使用... 传入参数依次进行绑定,多余的参数将以数组的形式交给变量 rest 如果没有参数传递给rest rest的会是空数组

变量作用域

  • 使用 var 申明变量实际上是有作用域的

  • 一个变量在函数体内声明这个变量在作用域为整个函数体,在函数体外是无法访问这个变量的

  • 如果两个不同的函数各自申请同一个变量,那么变量只在各自的函数体内起作用,不同函数内部的同名变量互相独立,互不影响

  • js 的函数 可以嵌套,此时函数可以访问外部函数定义的变量,反过来则不行

  • js 的函数在查找变量时从自身的函数定义开始,从内向外查找,如果内部函数定义了和外部函数重名的变量,则就近使用内部函数的变量

变量提升

  • 浏览器会对脚本进行预解析,所有的申明:包括变量声明和函数声明

  • 变量声明先将变量的初始值设置为undefined

  • 函数声明先将变量的初始值设置为函数初始字符串

  • 如果变量名和函数名相同,则优先函数

  • 函数内部定义变量时,要首先申请所有变量,最常用的做法是用一个var申明所有的变量

全局作用域

  • 不在任何函数内定义的变量就具有全局作用域

  • js默认的全局对象 window

  • 全局作用域的变量实际上绑定到到了window的一个属性

  • 顶层函数的定义也被视为一个全局变量并绑定到window对象

  • js 实际上只有一个全局作用域,任何变量或者函数 ,如果没有在当前的函数作用域中找到,就会继承往上查找,最后如果在全局作用域中也没有找到,就会报ReferenceError错误

名字空间

  • 全局变量会绑定到window是,不同js文件如果使用了相同的全局变量,或者定义了相同名字的顶层函数,都会造成命名冲突
  • 减少冲突的一个方法是将所有的变量和函数绑定到一个全局的变量的中

局部作用域

  • js的变量作用域实际上是函数内部
  • let 替代var可以申明一个块级作用域的变量
  • const 可以定义常量,和let一样,具有块级作用域,定义常量后,常量值无法被修改
var xm = {
    name : 'Google',
    birth : 1900,
    age : function () {
        var that = this;     //  that 保存对象的引用
        function getAgeFromBirth() {
            var y = new Date().getFullYear();
            return y - this.birth;   //  内部函数,this重新指向了window
        }
        return getAgeFromBirth();
    }

};

console.log(xm.age());
